Un archivo llamado CLAUDE.md con 8 reglas que le dicen a Claude: "deja de hablar de más". Es como ponerle un letrero a la maquinita que dice "solo dame lo que pedí, sin adornos". Lo instalas en 30 segundos y listo.

1 Lee antes de escribir. // Antes de tocar un archivo, Claude lo lee primero. Como cuando lees una receta COMPLETA antes de empezar a cocinar — así no te equivocas a la mitad.

Think before acting. Read existing files before writing code.

2 Sé conciso pero profundo // Claude te da la respuesta directa sin enrollarse. Piénsalo como un doctor: te dice qué tienes y qué tomar — no te cuenta la historia completa de la medicina.

Be concise in output but thorough in reasoning.

3 Edita, no reescribas // Si solo necesita cambiar UNA línea de código, Claude cambia esa línea — no te reescribe todo el archivo. Es como arreglar un botón suelto: solo coses ese botón, no haces la camisa de nuevo.

Prefer editing over rewriting whole files.

4 No releas lo que ya leíste // Si Claude ya leyó un archivo, no lo vuelve a leer a menos que haya cambiado. Es como ver un menú del restaurante: ya sabes qué hay, no lo vuelves a leer cada que pides algo.

Do not re-read files you have already read unless the file may have changed.

5 Prueba antes de decir listo // Claude verifica que el código funciona antes de decir que terminó. Como un chef que prueba el platillo antes de sacarlo de la cocina.

Test your code before declaring done.

6 Cero relleno y saludos // 
Nada de "¡Claro! ¡Excelente pregunta!" ni despedidas largas. Directo al grano. Imagina que pagas por cada palabra — porque así es: cada palabra te cuesta tokens.

No sycophantic openers or closing fluff.

7 Soluciones simples y directas // Claude no sobre-diseña cosas. Si necesitas un clavo, te da un martillo — no te construye una fábrica de martillos.

Keep solutions simple and direct.

8 Tú mandas // Si tú le pides algo específico, eso siempre gana sobre cualquier otra regla. Claude trabaja para ti, no al revés.

User instructions always override this file.

¿Dónde va el archivo?

Copia todo esto y pégalo en un archivo llamado CLAUDE.md en la raíz de tu proyecto. En la carpeta principal de tu proyecto — al mismo nivel donde tienes archivos como package.json o README.md. Claude Code lo detecta automáticamente cuando abres el proyecto.
